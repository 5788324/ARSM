# ARSM 项目交接文档

日期：2026-06-21 | 提交：13474d2 | 状态：Phase 6 已完成，进入 Phase 7 规划

---

## 当前状态

Phase 1-6 已完成。ARSM 现在已经具备比较完整的私人音频图书馆闭环：采集、导入、元数据补全、搜索、关系浏览、播放、字幕阅读/跟播、首页继续收听、个人评分、管理员元数据编辑、深色模式、启动器与 Docker。

### 已交付

- 采集：asmr.one Provider + runner + AcquisitionJob + 文件级进度可视化
- 导入：共享 import service + 重复检测 + 曲目分组 + 字幕识别
- 搜索：多字段搜索 + 排序 + 筛选 + 分页 + 地址栏状态同步
- 浏览：社团 / 声优 / 标签关系页
- 播放：全局播放器 + 队列 + 循环模式 + 倍速 + 续播
- 字幕：`.lrc/.vtt/.srt` 时间轴解析、跟播高亮、点击跳转、纯文本字幕阅读
- 首页：最近添加、最近采集、最近播放、统计、快捷入口
- 元数据：自动匹配 + 管理员手动编辑 + 手动重抓 asmr.one
- 用户数据：个人评分、收听历史、继续收听
- 部署：启动/停止批处理 + Docker 基础支持
- 质量：经过多轮审查修复，当前分支已完成 Phase 6 收口

### 当前限制

- 采集源仍以 asmr.one 为主，缺少第二来源
- 普通用户界面仍能看到管理员字段提示，后续可继续按角色隐藏
- 字幕目前支持跟播与阅读，但不支持翻译、双语对照、AI 生成
- 播放统计、最近播放数据已经起步，但还没有完整的“听了多少/最常听/最近完成”仪表能力
- 仍未提供 Android / 桌面正式打包壳

## 当前建议优先级

### P1：Phase 7 产品打磨
- 普通用户 / 管理员 UI 进一步分离，避免无权限区域出现在普通用户界面
- 作品详情页、首页、作品库继续做高频交互打磨
- 收听统计与继续收听体系深化
- Phase 6 新能力补测试与浏览器级验收

### P2：来源与生态
- 第二采集源预留或接入
- 字幕翻译、双语字幕、AI 字幕生成设计
- PWA / Android / 桌面壳方向评估
