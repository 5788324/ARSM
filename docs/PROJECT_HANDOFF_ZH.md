# ARSM 项目交接文档

日期：2026-06-20 | 提交：0a42931 | 状态：Phase 5 已完成，进入 Phase 6 规划

---

## 当前状态

Phase 1-5 已完成，v1.0 到 v1.1 前的主要产品化增强已经落地。当前版本已经具备“私人音频图书馆”的完整闭环：采集、导入、元数据补全、播放、搜索、关系浏览、字幕阅读、深色模式、启动器与 Docker。

### 已交付

- 采集：asmr.one Provider + runner + AcquisitionJob + 文件级进度可视化
- 导入：共享 import service + 重复检测 + 曲目分组 + 字幕识别
- 播放：全局播放器 + 队列 + 循环模式 + 倍速 + 续播
- 搜索：多字段搜索 + 排序 + 筛选 + 分页 + 地址栏状态同步
- 浏览：社团 / 声优 / 标签可点击 + 关系页
- 字幕：识别 + 文本阅读弹窗
- 主题：深色模式切换
- 元数据：采集后自动匹配 asmr.one 元数据
- 部署：启动/停止批处理 + Docker 基础支持
- 质量：已通过多轮代码审查，当前构建通过，测试基线 43/43

### 当前限制

- 采集源仍以 asmr.one 为主，缺少第二来源接入
- 字幕目前仅支持识别与阅读，不支持翻译、时间轴联动、AI 生成
- 作品详情页与播放器交互仍未完全对齐 asmr.one 的使用体验
- 仍缺评分、收听历史、播放统计、最近播放等产品层能力
- 仍未提供更像“桌面程序”的正式打包形态

---

## 当前建议优先级

### P1：Phase 6 产品增强
- 字幕增强：时间轴解析、跟播高亮、点击跳转、翻译入口
- 作品页与播放器 UI 重构：对标 asmr.one 的高频交互体验
- 收听体验：最近播放、继续收听、播放统计、队列体验增强
- 元数据增强：手动重抓、手动覆盖、封面/标签修正

### P2：来源与生态
- 第二采集源预留或接入
- Android / PWA / 桌面壳方向评估与最小实现
- 评分系统、评论/笔记、个人标记体系
