# ARSM 项目交接文档

日期：2026-06-21 | 提交：19c5058 | 状态：Phase 7 已完成，进入 Phase 8 规划

---

## 当前状态

Phase 1-7 已完成。ARSM 现在已经具备多用户结构下可长期使用的私人音频图书馆主链路：采集、导入、元数据补全、搜索、关系浏览、播放、字幕跟播、首页聚合、继续收听、个人评分、角色分离、基础统计、管理员元数据编辑、深色模式、启动器与 Docker。

### 已交付

- 采集：asmr.one Provider + runner + AcquisitionJob + 文件级进度可视化
- 导入：共享 import service + 重复检测 + 曲目分组 + 字幕识别
- 搜索：多字段搜索 + 排序 + 筛选 + 分页 + 地址栏状态同步
- 浏览：社团 / 声优 / 标签关系页
- 播放：全局播放器 + 队列 + 循环模式 + 倍速 + 续播
- 字幕：`.lrc/.vtt/.srt` 时间轴解析、跟播高亮、点击跳转、纯文本字幕阅读
- 首页：最近添加、最近采集、最近播放、统计、快捷入口
- 继续收听：真实统计口径的收听记录页
- 元数据：自动匹配 + 管理员手动编辑 + 手动重抓 asmr.one
- 用户数据：个人评分、收听历史、继续收听
- 角色分离：普通用户不再看到管理员字段与动作
- 测试：字幕解析新增测试，当前测试基线 51/51
- 部署：启动/停止批处理 + Docker 基础支持

### 当前限制

- 采集源仍以 asmr.one 为主，DLsite 仅为 stub，尚未真实接入
- 收听统计已经避免误导性指标，但仍未形成完整个人统计面板
- 字幕目前支持跟播与阅读，但不支持翻译、双语对照、AI 生成
- 尚无第二元数据来源
- 仍未提供 Android / 桌面正式打包壳

---

## 当前建议优先级

### P1：Phase 8 来源与能力扩展
- 第二来源或第二元数据来源真实接入
- 收听统计深化为更完整的个人面板
- Phase 6/7 能力补更系统的回归测试与浏览器验收

### P2：字幕与生态
- 字幕翻译、双语显示、AI 字幕生成设计
- PWA / Android / 桌面壳方案评估
