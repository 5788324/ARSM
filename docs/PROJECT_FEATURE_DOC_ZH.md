## 一、项目概览

ARSM 是一个面向私人长期使用场景的音频图书馆。当前保留多用户数据结构，但产品定位仍是私人项目优先：主要是你自己使用，同时不把系统写死成单用户。

| 项目 | 信息 |
|------|------|
| 仓库 | github.com/5788324/ARSM |
| 技术栈 | Next.js 16 + TypeScript + Prisma 5 + SQLite + NextAuth v5 + Tailwind 4 |
| 当前阶段 | Phase 9 已验收通过 |
| 当前重点 | Phase 10 产品收口与验收体系化 |

## 二、已实现功能

### 2.1 采集与导入

- asmr.one provider
- AcquisitionJob + runner
- inspect -> download -> import -> metadata 全链路
- 文件级下载进度
- 本地文件夹导入
- 重复检测与 review
- DLsite inspect-only provider

### 2.2 元数据

- 自动补全 asmr.one 元数据
- 圈子 / 声优 / 标签关联
- 管理员手动编辑标题、原标题、发售日
- 管理员手动重抓 asmr.one
- 多用户下个人评分独立存储

### 2.3 播放与字幕

- 全局播放器
- 队列 / 循环 / 倍速 / 续播
- 曲目自动分组
- 文本字幕阅读
- `.lrc/.vtt/.srt` 时间轴解析
- 跟播高亮与点击跳转

### 2.4 浏览与用户数据

- 作品库搜索、排序、筛选、分页
- URL 状态同步
- 高级筛选：排除词、时长、社团、声优、标签、带字幕
- 社团 / 声优 / 标签关系页
- 首页最近添加、最近采集、最近播放、统计
- 继续收听页
- 收藏
- 收听历史
- 个人评分（按用户独立）
- 普通用户 / 管理员界面分离
- 播放次数 / 收听作品数 / 总收听时长 / 最常播放

### 2.5 部署与运行

- Windows 启动 / 停止批处理
- Docker 基础支持
- 深色模式

## 三、真实验收结论

当前已确认通过的关键链路：

- `/works` 高级筛选面板真实可用
- 高级筛选输入值会同步到 URL
- `/continue` 页面可打开
- `PlayLog` 缺表问题存在明确修复路径
- “最常播放”在真实页面显示正确计数

---

## 四、当前限制

- 真实来源仍以 asmr.one 为主
- DLsite 仅为元数据级 provider
- 收听统计仍是基础版，不是完整个人中心
- 字幕暂不支持翻译、双语、AI 生成
- 尚无 Android / 桌面壳
