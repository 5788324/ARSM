重点确认：

- `/continue` 的“总收听时长”是否与真实播放一致
- 切歌、暂停、单曲循环、清空队列是否会漏记或重复累计

## 待验证路径

### 1. 播放 35 秒后暂停
步骤：
1. 登录测试账号
2. 打开任一可播放作品
3. 开始播放并持续约 35 秒
4. 点击暂停
5. 打开 `/continue`

预期：
- 总收听时长约增加 35 秒
- 不应出现明显翻倍或 0 秒不变

### 2. 播放 35 秒后切下一首
步骤：
1. 播放第一首约 35 秒
2. 点击下一首
3. 继续播放第二首几秒
4. 打开 `/continue`

预期：
- 第一首最后一段不丢失
- 第二首从 0 开始累计
- 不应把两首的累计混在同一次增量里

### 3. 单曲循环完整跑一轮
步骤：
1. 打开单曲循环模式
2. 让同一首完整播放到结束并自动重播
3. 再播放几秒后暂停
4. 打开 `/continue`

预期：
- 第一轮尾巴要被记录
- 重播后的累计从 0 重新开始
- 不应因为 `lastReportedSec` 残留导致前几十秒不计入

### 4. 清空队列前先播放一段
步骤：
1. 正常播放一段（例如 10-20 秒）
2. 点击清空队列
3. 打开 `/continue`

预期：
- 清空前最后一段会写入 `PlayLog`
- 不应因为直接停播而丢失最后一段
