# ARSM 下一阶段规划

日期：2026-06-20
定位：这是一份阶段路线说明，不是逐项执行清单。
执行请优先看 `docs/PHASE4_EXECUTION_TASKLIST_ZH.md`。

---

## 当前判断

ARSM 的核心采集链路已经打通，但产品体验还停留在 MVP：

- 采集有结果，但进度可视化弱
- 导入能完成，但曲目结构信息丢失
- 详情页能播，但播放器体验明显落后
- 元数据能抓，但还不是自动化流水线
- 字幕和台本还没有进入正式模型

因此下一阶段不是继续堆“更多功能”，而是先把最关键的使用体验补齐。

---

## 阶段目标

### Phase 4：可用性补全

重点解决：
- 采集进度可视化
- 曲目子文件夹分组
- 全局播放器基础版
- 元数据自动匹配
- 字幕 / 台本最小支撑

### Phase 5：产品化增强

在 Phase 4 稳定后再推进：
- 高级搜索与筛选
- VA / 社团联动浏览
- 评分系统
- 深色模式
- 更完整的播放队列与继续播放体验

### Phase 6：智能化与扩展

远期能力：
- AI 字幕
- 多 provider
- 更多平台适配
- 自动翻译 / 转录

---

## 竞品参考结论

参考对象：
- asmr.one
- neokikoeru
- kikoeru-express
- KikoFlu

吸收原则：
- 先学 asmr.one 的可用 UI 与播放器交互
- 再学 Kikoeru / KikoFlu 的文件树、字幕、队列、状态模型
- 不直接照搬全部复杂系统，优先做最小可用版本
